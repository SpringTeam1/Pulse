## 📌 프로젝트 개요

**Pulse**는 러닝을 즐기는 사용자들이 코스를 공유하고, 크루를 통해 소통하며, 개인 운동 기록을 체계적으로 관리할 수 있도록 설계된 웹 서비스입니다.
기존 JSP/Servlet 기반 "AllDayRun" 프로젝트를 **Spring MVC + Tiles 구조로 마이그레이션**하며, RESTful 아키텍처와 협업 중심 개발 방식을 도입한 팀 프로젝트입니다.

* **프로젝트 기간** : 2025.11.06 ~ 2025.11.17
* **개발 인원** : 5명
* **프로젝트 형태** : 팀 프로젝트 (JSP/Servlet → Spring MVC 마이그레이션)

### 🎯 핵심 목표

* 러닝 기록 · 코스 · 크루 · 챌린지를 하나의 서비스 흐름으로 통합
* 사용자 위치/데이터 기반 러닝 코스 추천 기능 구현
* 크루 단위 커뮤니티 활성화를 위한 전용 공간과 실시간 소통 제공
* Spring MVC 아키텍처와 REST API 기반 구조 정립
* Git 브랜치 전략을 활용한 협업 및 코드 품질 관리 경험

## 🧩 아키텍처 구조

```text
Client (JSP + Tailwind)
        ↓
Controller (Spring MVC)
        ↓
Service (비즈니스 로직)
        ↓
Repository / Mapper (MyBatis)
        ↓
Oracle DB (Autonomous DB)
```

* **Tiles 기반 공통 레이아웃** 적용 (헤더/푸터/사이드바 일원화)
* Controller → Service → DAO/Mapper → View 의 레이어드 아키텍처
* REST 스타일의 API + AJAX 기반 비동기 처리
* 공통 예외 처리 & 인터셉터를 통한 인증/인가 흐름 정리

## 📡 실시간 기능 구조 (SSE)

Pulse에서는 **크루 단체 톡 및 알림 기능**에 **SSE(Server-Sent Events)** 방식을 채택했습니다.

### 왜 SSE였는가?

* 채팅/알림처럼 **서버 → 클라이언트 단방향 실시간 전송**에 최적화
* WebSocket에 비해 구현 복잡도가 낮고, 운영/배포 환경에서 안정적
* HTTP 기반으로 동작해 프록시/방화벽 환경에서도 호환성 우수
* Polling/Long Polling 대비 네트워크 낭비를 줄이면서도 실시간성 확보

→ 크루 단체 채팅 메시지, 크루 활동 알림, 새로운 코스/챌린지 알림 등에 활용하여
사용자가 **페이지 새로고침 없이도 변화가 바로 반영되는 경험**을 제공했습니다.
